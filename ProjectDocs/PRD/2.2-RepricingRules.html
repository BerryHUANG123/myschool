调价规则设置页面

调价规则不包含商品最大值 、最小值的设定， 最大/小值的设置放在产品列表页便于批量设置。特别说明：最小价格的设置与卖家利益攸关，务必通过程序逻辑判定，明显不正常时应提醒卖家。

调价规则设置分为四个部分：

  • 智能调价设置: 未占据BuyBox时的规则设置，检测优先级：2级。
  • 黄金购物车设置: 占据BuyBox时的规则设置，检测优先级：2级。
  • 安全调价设置: 限制降价幅度，保证卖家利益，检测优先级：3级。
  • 全局设置: 无论是否占据BuyBox，都会起效的设置，检测的优先级：1级。

检测的逻辑从1级至3级。

说明：
竞争者：泛指除自己之外的其它所有卖家。但在应用黄金购物车设置时，竞争者是拥有BuyBox资格的其它卖家。
比价对象：特指按照竞争策略选出的要与之比价的卖家。

方框中的文字表示是下拉列表或复选框组件的选项，勾选的规则表示初始选中项。

规则判断顺序：
策略(Strategy) -> 条件(Condition) -> 状态(子条件，State) -> 规则(Rule)

智能调价设置

未占据BuyBox时使用的设置。

Part1-竞争策略选择

有2种竞争策略

  • 最低价格， 初始选项
    还需要选择竞争对手类型：
    • 亚马逊
    • FBA
    • FBM
    • 非特色卖家FBA
    • 非特色卖家FBM
  • 黄金购物车价格
    不需要选择竞争者类型。

Part2-调价条件设置

Strategy1-最低价格策略

比价对象默认是出价最低的竞争者，如果相应类型的高级设定开关开启，执行高级设置中的规则，不再执行Part2和Part4中的设置。

选择最低价格策略时，Part3中的设置无效，不显示。

说明: 以下条件互斥，只会执行其中某种情况的规则。

Condition1: 有任意竞争者在最小与最大价格之间
  • State1: 低价竞争者价格>最小价格
    • 低价竞争者价格【+/-】【数值/百分比】, 默认值是- 0.01$
    • 最小价格【+】【数值/百分比】, 默认值是+ 0.01$
    • 最大价格【-】【数值/百分比】, 默认值是- 0.01$

规则说明：
(1) 低价竞争者价格必定<最大价格；
(2) 低价竞争者是出价高于最小价格的卖家中的最低价者，而非所有卖家的最低价者；
(3) 当低价竞争者价格规则生效时，会自动跟随其价格来调价，有以下两种情况：

  • Case1: 低价竞争者价格<您的价格，降低价格
  • Case2: 低价竞争者价格>您的价格，抬高价格

业务用例： M1-S1-C1-State1-Case1
竞争者A, B, C, E：

  • 最小价格: $10.00
  • A: $11.00
  • B: $12.00
  • 您的价格：$13.00
  • C: $14.00
  • 最大价格: $15.00
  • E: $16.00

低价竞争者为A
调价结果：
A的价格低于您的价格，按照规则调整后价格为$10.99;


业务用例： M1-S1-C1-State1-Case2
竞争者A, B, C

  • 最小价格: $10.00
  • 您的价格：$11.00
  • A: $12.00
  • B: $13.00
  • 最大价格: $15.00
  • C: $16.00

低价竞争者为A
调价结果：
A的价格高于您的价格，按照规则调整后价格为$11.99;


  • State2: 最低出价<最小价格
    此处不会出现所有竞争者都低于或等于最小价格的情况
    • 自动竞争
    • 最小价格【+】【数值/百分比】, 默认值是+ 0.00$
    • 维持原价

规则说明：
(1) 比价对象默认是出价最低的竞争者，但是当自动竞争规则生效时，会自动选择出价高于最小价格的低价竞争者作为比价对象，并套用State1-"低价竞争者价格"的规则调整价格。
(2) 选择最小价格时，如果数值设置为0，表示使用最小价格，下同。
(3) 选择维持原价时不执行本次调价，下同。

业务用例： M1-S1-C1-State2
竞争者A, B, C, D, E：

  • A: $10.00
  • B: $11.00
  • 最小价格: $12.00
  • C: $13.00
  • 您的价格：$14.00
  • D: $15.00
  • 最大价格: $16.00
  • E: $17.00

最低出价者为A
调价结果：
当自动竞争规则生效时，低价竞争者是C，套用State2的规则，调整后价格为$12.99;


  • State3: 最低出价=最小价格
    规则及含义与State2相同，分别设置，默认规则也是自动竞争
Condition2: 无任何竞争者在最小与最大价格之间

说明最小价格和最大价格设置不合理，存在三种状态，可分别设置规则：

  • State1： 所有竞争者出价都≤最小价格

    • 维持原价
    • 最小价格【+】【数值/百分比】, 默认值是+ 0.01$
    • 最大价格【-】【数值/百分比】, 默认值是- 0.01$
    • 您的价格【+/-】【数值/百分比】, 默认值是- 0.01$

  • State2： 所有竞争者出价都>最大价格
    规则及含义与State1相同，分别设置，默认规则是最大价格

  • State3： 部分竞争者比最小价格低，但其它竞争者又比最大价格高
    规则及含义与State1相同，分别设置，默认规则是维持原价

业务用例： M1-S1-C2-State3
竞争者A, B, C, D：

  • A: $10.00
  • B: $11.00
  • 最小价格: $12.00
  • 您的价格：$13.00
  • 最大价格: $14.00
  • C: $15.00
  • D: $16.00

调价结果：由选中的规则决定


Strategy2-黄金购物车价格策略

比价对象黄金购物车拥有者，如果相应类型的高级设定开关开启，执行高级设置中的规则，不再执行Part2~Part4中的设置。

说明: 以下条件互斥，只会执行其中某种情况的规则。

Condition1: 购物车价格在最小与最大价格之间

最小价格≤购物车价格≤最大价格

  • State1: 购物车价格>最小价格
    • 黄金购物车价格【+/-】【数值/百分比】, 默认值是- 0.01$或-3%
    • 最小价格【+】【数值/百分比】, 默认值是+ 0.01$
    • 最大价格【-】【数值/百分比】, 默认值是- 0.01$

业务用例： M1-C1-S1
竞争者A, B, C, D：

  • A: $10.00
  • 最小价格: $11.00
  • B(购物车卖家): $12.00
  • 您的价格：$13.00
  • C: $14.00
  • 最大价格: $15.00
  • D: $16.00

调价结果：
当黄金购物车价格规则生效时，调整后价格为$11.99。


  • State2: 购物车卖家价格=最小价格
    • 维持原价
    • 最小价格【+】【数值/百分比】, 默认值是+ 0.00$
Condition2: 购物车价格不在调价范围内
  • State1: 购物车卖家价格<最小价格
    • 维持原价
    • 最小价格【+】【数值/百分比】, 默认值是+ 0.01$

业务用例： M1-C2-S1
竞争者A, B, C, D：

  • A: $10.00
  • B(购物车卖家): $11.00
  • 最小价格: $12.00
  • 您的价格：$13.00
  • C: $14.00
  • 最大价格: $15.00
  • D: $16.00

调价结果：
最小价格规则生效时, 调整后价格为$12.01;


  • State2: 购物车卖家价格>最大价格
    • 维持原价
    • 最大价格【-】【数值/百分比】, 默认值是- 0.00$

  • 维持原价

Part4-最小价格保障预设

如果调整后您的价格低于最小价格，为保障您的利益，按以下规则重新调整。

各开关默认为关闭状态。

黄金购物车设置

占据BuyBox时，唯一使用的设置。

Part1: 选择竞争者类型

  • FBA
  • FBM
  • 亚马逊

Part2: 黄金购物车设定

  • 调价策略：

    • 自动跟随最低价，保持竞争力
      • Condition1: 您的价格不是最低价
        "有BuyBox资格的最低出价"【-】【数值/百分比】, 默认是- 0.01$
      • Condition2: 您的价格是最低价
        "后一位有BuyBox资格的竞争者价格" 【-】【数值/百分比】, 默认是- 0.01$或- 3%
    • 提高我的黄金购物车价格，争取利润最大化
      "最有力竞争者价格"【-】【数值/百分比】, 默认是- 0.01$

    • 暂停黄金购物车设定

  • 最小与最大价格保障：

    • 当调整后的价格高于最大价格：
      • 维持原价

  • 策略3： 维持原价，不进行调价
  • 策略4： 无论您是否拥有黄金购物车，都会暂停黄金购物车设定，并套用"智能调价设置"。

安全调价设置

限制每次降价金额开关，开关默认开启

  • Condition: 当调价后的价格，低于你原价的 【折扣百分比】%, 默认值60%
    调整规则： 限制每次调价金额 = 你的价格 -【降价百分比】%, 默认值10%

业务用例： SafetyLimit
折扣百分比设为：60%
降价百分比设为：10%

Case1: 达到安全限制条件，启用限价
商品原价：$10.00
调整后的价格：$5.99, 降幅高于预设折扣百分比
调价结果：$9.00

Case1: 未达到安全限制条件，不启用限价
商品原价：$10.00
调整后的价格：$6.99, 降幅低于预设折扣百分比，不限价
结果：$6.99


全局规则

无竞争设置

当您是唯一卖家

  • 维持不变
  • 最大价格【-】【数值/百分比】, 默认值是- 0.00$表示使用最大价格
  • 您的价格【+/-】【数值/百分比】, 默认值是+ 0.00$
  • 停止调价

竞争者筛选条件设置

会优先筛选竞争者，再执行调价规则。
每个筛选项都是开关，开关均默认关闭，开关包括：

  • 卖家ID 【Include/Exclude】
  • 排除与亚马逊竞争
  • 排除 Backordered 的卖家
  • 排除 handling time 超过【时间段】的卖家，默认为2天
  • Free Shipping 的卖家 【Include/Exclude】
  • Expedited Shipping 的卖家 【Include/Exclude】
  • 排除店铺好评率低于【百分比】% 的卖家，默认值为0
  • 排除店铺评论量低于【数值】的卖家， 默认值为0
  • 选择要竞价的商品状态：
    • 一样状态，默认
    • 一样状态或更好
    • Used
      • Like New
      • Very Good
      • Good
      • Acceptable
    • Collectible
    • Refurbished

说明：
抢BuyBox时，可排除延期交货以避免故意降价

备注说明

与BQool规则设置不同处

除在布局上的不同，有以下差异：

智能调价设置差异
  1. 选择最低价格策略时:
  • 没有竞争者在您设定的最小最大价格之间，BQool只有一个条件，我们是分为3个条件（Strategy1-Condition2-State1~3)。
  1. 选择黄金购物车价格策略时:
  • 购物车价格低于或等于最小价格，BQool是分开设置条件的，我们合并为一个条件（Strategy2-Condition1)。
  • 购物车价格高于最大价格时，或低于最小价格，BQool将其合并为一个条件来设置，我们是分为两个条件（Strategy2-Condition1 and Condition3)。
黄金购物车设置差异
  • BQool黄金购物车设置中含有安全调价，智能调价设定中也含有安全调价，两者设置内容一致，我们将其合并后为通用设置。

需技术研究的接口问题

  • 能够区分有购物车资格的卖家，主要是特色卖家和非特色卖家;
